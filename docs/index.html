Agent Delivery

Tell the agent where to start.

Choose A Setup Mode

Reuse an existing local install when it exists. Clone once for a first-time personal setup. Use a repo-pinned dependency when a team needs the same reviewed version.

Read The Project First

Local prompts should tell Codex, Claude, Antigravity, or another agent to read the current project's AGENTS.md, CLAUDE.md, CODEX.md, .agents/README.md, task docs, and PRD/ARD docs before applying shared guidance.

PRD Has Its Own Route

Use the prd route when the deliverable is a product requirements note. Use the product route only when the task continues from PRD into ARD, code, tests, UI checks, and commit readiness.

Show Gate Signals

Every completed gate or task step should emit a traffic-light signal with the gate name, evidence, and next gate. Completion requires every required gate to be GREEN. 

Choose setup mode first:
1. Existing local install: use AGENTPLAYBOOK_HOME or the path I provide.
2. First-time local shared install: clone once to ~/.agent-playbook.
3. Team-pinned install: use a repo-pinned submodule or dependency.

Validate the selected root:
python3 ${AGENTPLAYBOOK_HOME}/scripts/workflow.py validate

Use this project's current agent instructions first:
AGENTS.md, AGENTS.override.md, CLAUDE.md, CODEX.md, .agents/README.md,
CONTRIBUTING.md, task docs, PRD/ARD docs, or equivalent project docs.

Then use AgentPlaybook:
${AGENTPLAYBOOK_HOME}/AGENTS.md
${AGENTPLAYBOOK_HOME}/index.md
${AGENTPLAYBOOK_HOME}/scripts/workflow.py

PRD only:
python3 ${AGENTPLAYBOOK_HOME}/scripts/workflow.py route prd --platform web --concern ui

After each completed gate:
Gate signal: GREEN | gate: <gate> | evidence: <evidence> | next: <next gate>
