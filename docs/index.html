Routing

The playbook is loaded selectively.

Agents should not read everything by default. They classify the request, route to the smallest useful document set, then execute the route gates with evidence.

01

Read local rules

Project instructions remain the source of truth for commands and domain policy.

02

Select route

Use the workflow router for feature, docs, review, release, PRD, or debugging work.

03

Load cards

Read only the common, platform, workflow, and product cards needed for the task.

04

Report evidence

Completion requires concrete verification or a clear blocked reason.
